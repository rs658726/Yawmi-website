Privacy Policy & Terms Of Use

General

The policy below applies to all users of the Yawmi application for MacOS/iOS (also referred to as "our app").

We do not collect, store, or share any personal information related to our users. We do not have access to user data.

Our app allows you to integrate third party applications into your workflow such as Google or Github. For more information on how your personal information is used by a third party, please refer to the third party's privacy policy and terms of use.

By using our app, you consent to our privacy policy and agree to its Terms and Conditions.

We reserve the right to make changes to this policy. Thus, we encourage you to periodically review this page for any changes. We will notify you of any changes by posting the new Privacy Policy on this page. These changes are effective immediately after they are posted on this page.

Gmail Integration

Authentication and Authorization

We implemented the Gmail OAuth 2.0 flow to authenticate and authorize Gmail-related transactions as defined by the Google API documentation.

Scopes

In order for our user to be able to search, view, edit, or delete their messages, we request the minimum level of access needed which is, in this case, the modify scope, and is defined as follows:

All read/write operations except immediate, permanent deletion of threads and messages, bypassing Trash.

For more information about the Gmail API scopes, click here.

Data Storage

We do not store, share, or collect your Gmail messages or your personal information.

We only store two types of data that are vital to the functioning of Yawmi:

  • Authentication/Authorization tokens are stored in your private Keychain and are needed to log you in automatically when you launch Yawmi. This is all managed by the official Google Sign In SDK for iOS.
  • The identification number of any Gmail message you added as a Yawmi task is stored in your private database in order for you to access that message directly from the task detail page.

Yawmi's use and transfer to any other app of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements.

Github Integration

Authentication and Authorization

We implemented the Github OAuth 2.0 flow to authenticate and authorize Github-related transactions as defined by the Github API documentation.

Scopes

In order for our user to be able to query and view pull requests and issues, we request the type of access known as repo and defined as follows:

Grants full access to public and private repositories including read and write access to code, commit statuses, repository invitations, collaborators, deployment statuses, and repository webhooks. Note: In addition to repository related resources, the repo scope also grants access to manage organization-owned resources including projects, invitations, team memberships and webhooks. This scope also grants the ability to manage projects owned by users.

Please note that the Github API doesn't offer granular levels of access like other APIs such as Google do, hence the access level discrepancy between what is requested and what is needed.

For more information about the Github API scopes, click here.

Data Storage

We do not store, share, or collect your pull requests, issues, or personal information.

We only store two types of data that are vital to the functioning of Yawmi:

  • Authentication/Authorization tokens are stored in your private Keychain and are needed to log you in automatically when you launch Yawmi.
  • The number of the pull request or issue you added as a Yawmi task is stored in your private database in order for you to access that PR or issue directly from the task detail page.
